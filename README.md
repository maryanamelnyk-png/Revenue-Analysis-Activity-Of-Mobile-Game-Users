
# Mobile Gaming Product Analytics Dashboard

## Project Overview
This project focuses on analyzing user activity and revenue in a mobile gaming product.

The goal was to transform raw user event data into actionable product metrics and insights to support business decision-making.


## Key Questions
- How does player activity change over time?
- What percentage of users engage with Battle Pass?
- How much time do players spend in the game across age groups?
- How does revenue evolve over time?
- What trends can be observed through cohort analysis?


## What Was Done
- Calculated unique users and Battle Pass engagement rate
- Analyzed average playtime per user (HH:MM format)
- Segmented users into age groups (5-year intervals)
- Built heatmap of player activity by age and time
- Calculated New MRR based on first payment
- Analyzed revenue trends and month-over-month changes
- Built cohort analysis with revenue retention over time


## Key Insights
- Battle Pass engagement decreased from ~76% to ~50% → potential loss of interest
- Active users increased significantly → product growth with declining feature engagement
- Playtime fluctuates (from ~53 to ~78 hours/month) → possible seasonality or in-game events impact
- Younger users (15–24) are the most engaged
- New MRR shows high volatility → нестабільне залучення платних користувачів
- Revenue trends are inconsistent with sharp drops and recoveries
- Cohort analysis shows typical revenue decay, but some cohorts perform better → potential high-quality acquisition channels


## Recommendations
- Investigate decline in Battle Pass engagement (UX, rewards, relevance)
- Analyze acquisition channels to identify high-performing cohorts
- Explore drivers of revenue volatility (campaigns, events, pricing)
- Focus on retention strategies for high-value user segments


##  Tools & Technologies
- Tableau (dashboards & visualization)
- LOD expressions (Level of Detail calculations)
- Table Calculations (dynamic metrics)
- Calculated fields (segmentation & custom metrics)
